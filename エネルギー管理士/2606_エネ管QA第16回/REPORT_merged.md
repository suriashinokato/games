## 2. 🔴 最優先修正項目

### #1 — Q2 / p.102 左段「1. テブナンの定理のおさらい」末尾　【発見元：Codex】
- **誤**: 「ただし、交流回路においては、**抵抗ではなくインピーダンス**が登場し、電圧・電流は大きさと位相をもつフェーザを扱う点が異なります。」
- **正**: 「ただし、交流回路においては、**抵抗だけでなくインピーダンス**が登場し、電圧・電流は大きさと位相をもつフェーザ **として** 扱う点が異なります。」
- **理由**: 図 2 以降の交流回路でも負荷は抵抗 R。「抵抗ではなく」と書かれると **抵抗を扱わない** ように読めて技術的誤読を招く。「抵抗だけでなく」が正確。
- **観点**: 7. ファクト誤り（説明整合）／11. 表現精度
- ※ Claude Code 側はこの誤読リスクを見落とした。Codex の重要な拾い。

### #2 — Q1 / p.100, 101「漏洩」/「漏えい」混在　【発見元：両方】
- **誤（混在）**:
  - p.100 1.(2): 「外部に **漏洩** した場合は」
  - p.101 2 末尾: 「**情報漏洩**、ファイル改ざん、システム停止」
  - p.101 3 冒頭: 「**漏えい**、改ざん、消失、不正利用」
- **正（統一）**: すべて「**漏えい**」に統一（「外部に漏えいした場合は」「情報漏えい」）。
- **理由**: 「洩」は常用漢字外。新聞用字用語ハンドブック・公用文表記ガイド・JIS Q 27000 系本文・GLOSSARY 推奨ともに「漏えい」が標準。
- **観点**: 3. 表記ゆれ
- ※ 両者が独立検出。**相互担保された修正**。

### #3 — Q1 / p.100, 101「ウィルス」/「ウイルス」　【発見元：両方】
- **誤**:
  - p.100 完全性: 「**コンピュータウィルス** によって変更・削除」
  - p.100 完全性: 「**コンピュータウィルス対策**」
  - p.101 2 末尾: 「**ウィルス感染** では」
- **正**: 「**コンピュータウイルス**」「**ウイルス感染**」に統一。
- **理由**: JIS X 0001、IPA「コンピュータウイルス対策基準」、文部科学省検定教科書、新聞用語委員会いずれも「ウイルス」が標準。原稿内では「ウィルス」で統一はとれているが、p.101 4.④ の「マルウェア対策」と並記される文脈で外部標準寄せが自然。
- **観点**: 2. 用語ゆれ
- ※ 両者が独立検出。**相互担保された修正**。

## 3. 🟡 編集判断項目

### #4 — Q1 / p.101 3「狭義的には」　【発見元：Claude Code】
- 「**狭義的には**」→「**狭義には**」へ。
- 理由: 「狭義 + 的」の派生形は稀。並列概念「広義には〜、狭義には〜」と対をなす標準形。

### #5 — Q1 / p.100 2.(1) 機密性「言い換えると、〜ともいえます」　【発見元：Claude Code】
- 「**言い換えると、『見てはいけない人に見せないこと』ともいえます**」
- 「言い換える」と「〜ともいえる」が機能重複。どちらかに統一。
- 観点: 11. 冗長

### #6 — Q1 / p.100 1.(1) 情報資産の定義　【発見元：Claude Code】
- 「ヒト・モノ・カネに関わる情報やデータ」 → JIS Q 27000:2019 の「組織にとって価値のある情報」に観点を寄せた表現を推奨。
- 推奨: 「企業や組織にとって価値のある情報やデータなど」。
- 観点: 7. ファクト精度／11. 表現

### #7 — Q1 / p.100 1.(2)「セキュリティとは、安全性を確保するための概念」　【発見元：Claude Code】
- 「**概念**」→「**取り組み**」「**考え方**」へ寄せると導入として自然。
- 観点: 11. 表現（任意）

### #8 — Q2 / p.102 右段「電圧 Ė[V] の定電圧源」　【発見元：Codex】
- 「**定電圧源**」→「**フェーザ電圧 Ė[V] の交流電圧源**」へ寄せる案。
- 理由: 「定電圧源」は誤りではないが、直流の定電圧源と誤読されやすい。交流回路の文脈では「交流電圧源」のほうが明確。
- 観点: 11. 初学者向け説明

### #9 — Q2 / p.103 図 4・図 5・式中の Ż₀ のドット表記方針　【発見元：Codex】
- 等価インピーダンス Ż₀ にドットが付いている。
- 本文では「電圧・電流はフェーザを扱う」と説明している一方、インピーダンスにもドットがあるため、編集方針の確認が必要。
- 「複素インピーダンスにもドットを付ける」方針なら現状維持。「電圧・電流のみドット付き」方針なら Ż₀ → Z₀ に統一要。
- 観点: 4. 数式記号の整合性

### #10 — Q2 / p.103 (2) 並列合成の分母 jX₁ − jX₃ への注記　【発見元：Claude Code】
- (jX₁) + (−jX₃) = jX₁ − jX₃ の対応を一行注記すると初学者に親切。
