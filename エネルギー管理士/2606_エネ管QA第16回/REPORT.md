## 3. 🟡 中優先（編集判断）

### #2 — Q1 / p.100, 101 「コンピュータウィルス」/「ウィルス」
- 原稿内では「ウィルス」で 3 か所統一されているが、JIS X 0001、IPA「コンピュータウイルス対策基準」、文部科学省検定教科書、新聞用語委員会いずれも**「ウイルス」が標準**。
- 「マルウェア対策」（p.101 4.④）と並記される文脈で外部標準寄せが自然。
- **推奨**: 「コンピュータウイルス」「ウイルス」に統一（または編集方針として原稿表記を確定）。

### #3 — Q1 / p.101 3「狭義的には」→「狭義には」
- 「狭義 + 的」の派生形は稀。並列概念「広義には〜、狭義には〜」と対をなす標準形へ。

### #4 — Q1 / p.100 2.(1)「言い換えると、〜ともいえます」が機能重複
- 「言い換える」と「〜ともいえる」が二重。どちらか一方で十分。

### #5 — Q1 / p.100 1.(1) 情報資産の定義「ヒト・モノ・カネに関わる情報」
- 経営三資源フレームに寄せた説明だが、JIS Q 27000:2019 の「組織にとって価値のある情報」とは観点が異なり、続く例示（顧客情報・財務情報・技術情報・記録）と整合性が弱い。
- **推奨**: 「企業や組織にとって価値のある情報やデータなど」へ。

> 詳細はすべて [corrections/Q1_情報セキュリティ.md](corrections/Q1_情報セキュリティ.md) 参照。

## 4. 🟢 軽微な気付き（修正必須ではない）

### #6 — Q1 / p.100「セキュリティとは、安全性を確保するための **概念** です」
- 「概念」より「取り組み」「考え方」のほうが導入として自然。文体の好み。

### #7 — Q2 / p.103 (2)「並列合成の分母 jX₁ − jX₃」の注記
- 計算的に正しいが、`(jX₁) + (−jX₃) = jX₁ − jX₃` の対応を一行注記すると初学者に親切。

### #8 — Q2 / p.103 (3) İ_R 分母虚部「X₃X₁」の表記順
- `X₁X₂ − X₂X₃ − X₃X₁` の最終項のみ降順。`X₁X₃` に揃えると添字昇順で揃う。等価。

> 詳細は [corrections/Q2_テブナンの定理.md](corrections/Q2_テブナンの定理.md) 参照。
