# Q1 校正結果 — 「情報セキュリティって？」

- 対象: p.100–101（2 ページ、2 段組）
- 課目区分: Ⅱ電気の基礎 — 自動制御及び情報処理
- 担当キャラ: なべさん

## 1. 全体所見

情報セキュリティの 3 要素（CIA：機密性・完全性・可用性）と情報セキュリティポリシーの階層構造（基本方針 / 対策基準 / 実施手順）の説明は、JIS Q 27000 系および IPA の解説と整合しており、初学者向け入門として技術的に妥当。ただし**章をまたいだ表記ゆれ**（漏洩 / 漏えい、ウィルス / ウイルス）と、軽微な日本語表現の冗長・不自然箇所が複数ある。

## 2. 🔴 表記ゆれ（章内クロスチェック）

### #Q1-1 「漏洩」/「漏えい」の混在　【観点：表記ゆれ／用字統一】
- p.100 1.(2) 情報セキュリティとは: 「仮に情報資産が外部に **漏洩** した場合は」
- p.101 2. 末尾: 「ウィルス感染では、**情報漏洩**、ファイル改ざん、システム停止」
- p.101 3. 情報セキュリティポリシー: 「**漏えい**、改ざん、消失、不正利用などの脅威」

**推奨**: 「漏えい」に統一。
**理由**: 「洩」は常用漢字外。新聞用字用語ハンドブック（共同通信／朝日／毎日）、公用文表記ガイド、および JIS Q 27000 系の本文（「情報漏えい」）でも「漏えい」交ぜ書き表記が標準。本誌ほかの電気・エネ管解説は「漏えい」が多数。同一記事内で 3 か所中 1 か所だけ「漏洩」となっており修正コストも低い。

### #Q1-2 「コンピュータウィルス」/「ウィルス」　【観点：用語表記／JIS 整合】
- p.100 2.(2) 完全性: 「コンピュータ**ウィルス**によって変更・削除」
- p.100 2.(2) 完全性: 「**コンピュータウィルス対策**」
- p.101 2. 末尾: 「**ウィルス**感染では」

**推奨**: 「コンピュータウイルス」「ウイルス」に統一。
**理由**: JIS X 0001:1994（情報処理用語）、IPA「コンピュータウイルス対策基準」（経済産業省告示）、文部科学省検定教科書、新聞用語委員会の用字用語ハンドブック、および本誌でも近年は「ウイルス」（小書きの「ィ」ではなく大きい「イ」）が標準。原稿内では「ウィルス」で統一はとれているが、「マルウェア対策」（p.101 4.④）と並記される文脈で外部標準に合わせる方が自然。

> 編集判断で原稿内統一（「ウィルス」存置）を選ぶ場合は、誌全体の用字方針として確定しておきたい。

## 3. 🟡 日本語表現・冗長

### #Q1-3 「狭義的には」　【観点：日本語表現】
- p.101 3. 情報セキュリティポリシー: 「このうち、**狭義的には** 基本方針と対策基準を合わせて情報セキュリティポリシーといいます。」

**推奨**: 「**狭義には**」へ。
**理由**: 「狭義 + 的」の派生はやや稀。広辞苑・大辞林・現代用例ともに副詞的用法は「狭義には」「広義には」が標準。並列概念の「広義には〜、狭義には〜」と対をなす用法でもある。

### #Q1-4 「言い換えると、〜ともいえます」が二重　【観点：冗長】
- p.100 2.(1) 機密性: 「言い換えると、『見てはいけない人に見せないこと』**ともいえます**。」

**推奨**: 「言い換えると、『見てはいけない人に見せないこと』**です**。」
または: 「『見てはいけない人に見せないこと』**ともいえます**。」（冒頭の「言い換えると」を削除）
**理由**: 「言い換える」と「〜ともいえる」が機能重複。どちらか一方で十分。

## 4. 🟡 用語の精度（編集判断）

### #Q1-5 「情報資産」の定義表現　【観点：ファクト／用語精度】
- p.100 1.(1) 情報資産とは: 「情報資産とは、企業や組織が保有する **ヒト・モノ・カネに関わる情報やデータなど** のことです。」

**気付き**: 「ヒト・モノ・カネ」は経営三資源（経営学のフレーム）であり、JIS Q 27000:2019 の定義「**組織にとって価値のある情報・知識・媒体・処理能力等**」とは観点が異なる。「ヒト・モノ・カネに関わる情報」と限定すると、経営三資源外の情報（研究データ、設計図、ログ、知的財産など）が漏れているように読まれかねない。

**推奨**: 「情報資産とは、企業や組織にとって価値のある情報やデータなどのことです。具体的には、顧客情報や財務情報、技術情報、記録などが含まれます。」
**理由**: JIS / IPA の定義に近づけつつ、続く例示（顧客情報・財務情報・技術情報・記録）と整合する。エネ管試験ではこの種の出題で JIS 系の用語定義が出ることがあり、原典寄りの表現にしておく方が安全。

### #Q1-6 「セキュリティとは、安全性を確保するための概念です」　【観点：表現】
- p.100 1.(2) 情報セキュリティとは: 冒頭一文。

**気付き**: 「概念」よりも「考え方」「取り組み」「状態」のほうが、続く「**情報セキュリティ**」（情報を守ること）への導入として自然。「概念」は抽象度が高すぎ、対比対象が曖昧。

**推奨（任意）**: 「セキュリティとは、安全性を確保するための **取り組み** のことです。」
**判断**: 文体的好みの範囲。修正必須ではない。

## 5. ✅ 確認済み（問題なし）

- 機密性 (confidentiality) / 完全性 (integrity) / 可用性 (availability) の英語表記・対応・説明 — JIS Q 27000:2019 と整合。
- 情報セキュリティポリシーの 3 階層（基本方針 / 対策基準 / 実施手順）— IPA「中小企業の情報セキュリティ対策ガイドライン」等の慣習表現と整合。
- 図 1 のピラミッド階層図（基本方針が最上位）— 正しい向き。
- p.101 4.「ユーザ認証 / アクセス制御 / 暗号化 / マルウェア対策」— エネルギー管理士試験（課目II 自動制御及び情報処理）で実際に出題されている対策カテゴリと一致。
- 課目区分「Ⅱ電気の基礎 — 自動制御及び情報処理」の表記 — エネ管試験要綱と一致。
